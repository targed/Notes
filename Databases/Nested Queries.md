- A nested query is a complete `SELECT` statement (inner query) embedded inside the `WHERE` clause of another query (outer query).
- #### **1. Uncorrelated Nested Queries**
  In an uncorrelated query, the inner query **does not** depend on the outer query. It can be run independently.
  
  *   **Evaluation Order:**
    1.  The **inner query** runs first (once).
    2.  It passes its result up to the **outer query**.
    3.  The **outer query** runs using that result.
  
  *   **Equality Checks (`=`):**
    *   If you use `=`, the inner query must return **exactly one value** (a single cell).
    *   *Example:* `WHERE R.sid = (SELECT sid FROM Sailor WHERE sname='George')`.
  
  *   **Set Checks (`IN` / `NOT IN`):**
    *   If the inner query returns a list of values (multiple rows), you must use `IN` or `NOT IN`.
    *   *Example:* `WHERE S.sid IN (SELECT R.sid ...)` finds sailors whose ID appears in the list generated by the subquery.
  
  *   **Set Comparison (`ANY` / `ALL`):**
    *   `> ANY`: Greater than *at least one* value in the subquery result (equivalent to `> MIN`).
    *   `> ALL`: Greater than *every* value in the subquery result (equivalent to `> MAX`).
  
  *   **Multi-Level Nesting:**
    *   You can nest queries inside other nested queries.
    *   The example shows finding "Names of sailors who did **not** reserve a **red** boat".
    *   Logic: Find Red Boats -> Find IDs who reserved them -> Find Sailors whose ID is `NOT IN` that list.
  
  ---
- #### **2. Correlated Nested Queries (CNQ)**
  A correlated subquery is much more complex because the inner query **references an attribute from the outer query**. The two are linked.
  
  *   **Execution Logic (Looping):**
    *   It acts like a **nested for-loop** in programming.
    *   The system picks the **first row** of the Outer Query.
    *   It runs the Inner Query using values from that specific outer row.
    *   It evaluates the condition.
    *   It moves to the **second row** of the Outer Query and repeats.
  
  *   **The `EXISTS` and `NOT EXISTS` Operators:**
    *   These are designed specifically for correlated queries.
    *   **`EXISTS`**: Returns `TRUE` if the subquery returns **at least one row**. It checks for "non-empty" results.
    *   **`NOT EXISTS`**: Returns `TRUE` if the subquery returns **zero rows** (empty set).
    *   *Example:* Find employees with dependents.
        ```sql
        SELECT * FROM Employee E
        WHERE EXISTS (SELECT * FROM Dependent D WHERE E.ssn = D.essn);
        ```
        *   For every employee `E`, SQL checks if there is a matching dependent `D`.
  
  *   **Performance Note:** Correlated queries can be slow because the inner query executes repeatedly. Sometimes they can be rewritten as `JOIN`s for better performance.
  
  ---
- #### **3. Relational Division in SQL**
  SQL does not have a `DIVIDE BY` operator. To perform "Division" (e.g., "Find sailors who have reserved **ALL** boats"), you must use logic based on **Double Negation**.
  
  *   **The Logic:**
    *   Instead of saying: "Find sailors who reserved every boat."
    *   SQL says: "Find sailors such that **there does not exist** a boat that they have **not reserved**."
  
  *   **The Query Structure:**
    *   Outer Query: Select Sailor `S`.
    *   Middle Query: `WHERE NOT EXISTS` (Check for a "bad" boat).
    *   Inner Query: A boat `B` that Sailor `S` has `NOT` reserved.
